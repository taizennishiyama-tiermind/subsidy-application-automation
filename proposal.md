## 1. 本提案の要点

補助金申請書の作成業務を AI で自動化・効率化する**専用環境を構築し、貴社に納品**するサービスです。

### ひとことで言うと

> **「補助金申請のプロ知識を持つ AI アシスタントを、貴社専用にセットアップしてお届けします」**

### 3つのポイント

| | ポイント | 詳細 |
|---|---|---|
| **① 買い切り型** | 一度の構築費用で永続的に利用可能 | 月額費用なし（Claude Code のサブスクリプションのみ） |
| **② 完全自走型** | 納品後は貴社スタッフだけで運用可能 | コマンド1つでヒアリング〜申請書完成まで |
| **③ 拡張性あり** | 新しい補助金にも自分で対応可能 | `/add-grant` コマンドでいつでも追加 |

## 2. 現状の課題と解決策

### 補助金申請業務の典型的な課題

多くの補助金申請代行事業者が抱える共通の課題です。

```
【現状】
  ① 担当者の経験・知識に依存 → 品質にばらつき
  ② ヒアリングから申請書作成まで 1案件あたり10〜20時間
  ③ 補助金ごとに書式・審査基準が異なり、都度調査が必要
  ④ テンプレートへの転記ミスが発生しやすい
  ⑤ 担当者が退職すると知識・ノウハウが失われる
```

### 本サービスが実現すること

```
【導入後】
  ① AI がプロの手法でヒアリング → 誰が担当しても安定した品質
  ② 1案件あたりの工数を 約60〜70% 削減（→ 3〜6時間程度）
  ③ 4補助金の審査基準・書式をAIが記憶 → 調査不要
  ④ テンプレートへの自動書き込み → 転記ミスゼロ
  ⑤ ナレッジは環境内に蓄積 → 属人化を排除
```

## 3. サービス概要・できること

### 対応補助金（初期設定）

| 補助金 | 補助上限（概算） |
|--------|-----------------|
| ものづくり補助金 | 750万〜4,000万円 |
| IT導入補助金 / デジタル化・AI導入補助金 | 5万〜3,000万円 |
| 新事業進出補助金（旧: 事業再構築補助金） | 2,500万〜7,000万円 |
| 小規模事業者持続化補助金 | 50万〜200万円 |

※ `/add-grant` コマンドでいつでも追加可能。

### 業務別の Before / After

| 業務 | 導入前（従来） | 導入後 |
|------|--------------|--------|
| ヒアリングシート作成 | 補助金ごとに手作業で作成（30〜60分） | 補助金情報を入力するだけで自動生成 |
| クライアントヒアリング | メモを取りながら手動整理（2〜3時間） | AI が1問ずつ質問し、自動でデータ整理 |
| 財務データの分析 | 決算書を目視で読み取り（1〜2時間） | PDF/Excel を読み込ませて自動分析（数分） |
| 最適な補助金の選定 | 経験と勘に依存 | 事業内容・財務状況から AI がスコアリング |
| 申請書ドラフト作成 | 一から手書き・転記（3〜5時間） | 審査基準を踏まえた申請書を AI が自動生成 |
| テンプレートへの記入 | 手動で Excel/Word に転記（1〜2時間） | 公式テンプレートへ自動書き込み（数分） |
| セルフレビュー | 経験者によるチェック（1〜2時間） | 審査基準に照らした品質チェックを AI が実施 |

### ワークフロー

```
┌──────────────────────────────────────────────────────────────┐
│                         操作の流れ                            │
├──────────────┬───────────────────────────────────────────────┤
│  コマンド     │  AI が実行すること                             │
├──────────────┼───────────────────────────────────────────────┤
│ /new-case    │  案件フォルダ作成 + ヒアリング開始              │
│ /hearing     │  中断したヒアリングを再開                       │
│ /analyze     │  決算書を読み込み財務指標を自動算出             │
│ /match       │  6軸スコアリングで最適補助金を提案             │
│ /draft       │  審査基準準拠の申請書ドラフトを自動生成         │
│ /review      │  採択基準でセルフレビュー・改善提案             │
│ /fill        │  公式 Excel/Word テンプレートに自動書き込み     │
└──────────────┴───────────────────────────────────────────────┘
                              ↓
                    申請書の完成・提出へ
```

## 5. サービスの進め方

### STEP 1: 対面ヒアリング・設計（実装費用に含む）

貴社事務所に伺い、以下を確認します。

- 現在の補助金申請業務フロー
- 使用しているテンプレート・書式
- よく申請する補助金の種類
- クライアントへのヒアリング方法・質問パターン
- 自動化によって解決したい課題・優先事項
- 自社独自のノウハウ・採択実績

ヒアリング内容を反映し、**貴社の業務フローに最適化した環境**を構築します。

---

### STEP 2: AI 自動化環境の構築（実装 約2時間）

ヒアリング内容を踏まえ、貴社専用の AI 環境を構築します。

**構築内容（標準）:**

- Claude Code 上で動作する補助金申請支援環境一式
- 対応補助金の設定（初期4種類、追加可能）
- 補助金ごとの制度概要・審査基準のナレッジベース
- 貴社のヒアリング手法・質問パターンの組み込み
- 財務データ自動分析機能（決算書 PDF 対応）
- 申請書自動生成機能（ストーリーテリング対応）
- Excel/Word テンプレートへの自動書き込み機能
- 審査基準に基づくセルフレビュー機能
- スラッシュコマンドによる直感的な操作体系
- 貴社独自ノウハウの初期登録（採択実績・業界別ポイント等）

---

### STEP 3: 納品物の引き渡し

以下一式を納品します。

| 納品物 | 内容 |
|--------|------|
| AI 自動化環境一式 | 上記構築内容のすべてのファイル・設定（zip 形式） |
| 運用マニュアル | セットアップ手順、操作方法、トラブルシューティング |
| カスタマイズガイド | 補助金の追加方法、テンプレートの更新方法、ノウハウ追加方法 |

---

### STEP 4: 操作レクチャー（約1時間）

納品後、以下の内容でレクチャーを実施します。

- Claude Code の基本操作
- 実際の案件を使った一連のワークフロー体験
- カスタマイズ方法（補助金追加、テンプレート更新、ノウハウ追加）
- 質疑応答

**完了基準:** 貴社ご担当者様が本ツールを使って一通り申請書を作成できることを確認した時点で、本案件をクローズとします。

## 11. 本サービスの特長

### 「申請書代行」との違い

| 比較軸 | 申請書代行 | 本サービス（AI 環境構築） |
|--------|-----------|------------------------|
| コスト | 1件ごとに費用発生 | 構築費用のみ（使い放題） |
| 品質の安定性 | 担当者に依存 | 常に審査基準準拠の品質 |
| スピード | 数日〜数週間 | 数時間で申請書ドラフト完成 |
| 拡張性 | 依頼のたびに発注 | 自社で新補助金を追加可能 |
| 知識の蓄積 | 外部業者に蓄積 | 自社に蓄積（ノウハウ資産化） |
| 属人化リスク | 高い | 低い（誰でも同品質） |

### なぜ今 AI なのか

2026年現在、AI の文章生成精度は急速に向上しています。一方、補助金申請業務の本質（クライアントの強みを引き出す、審査基準に沿ったストーリーを構築する）は変わりません。
本サービスでは、**AI の得意なことと人間の判断が必要なこと**を明確に分け、両者を組み合わせた最適なワークフローを実現します。

### 対応補助金の拡張性

初期設定の4補助金に限らず、`/add-grant` コマンドで**いつでも新しい補助金を追加**できます。補助金の概要情報を渡すだけで、同じワークフローで申請書作成が可能になります。

## 12. 守秘義務・セキュリティ

### 情報の取り扱い

- 本サービス実施にあたりヒアリングする貴社の業務情報・ノウハウは、本サービス提供の目的以外に使用しません
- 貴社のクライアント情報（決算書等）はローカル環境で処理されます。外部サーバーへの情報送信は Claude API を通じた処理のみです
- Claude API の利用規約に基づき、入力データはモデルの学習に使用されません（API 利用の場合）

### セキュリティに関する注意事項

- 本環境はローカル PC で動作します。クラウドへのデータ保存は行いません
- 決算書等の機密情報を扱う際は、貴社のセキュリティポリシーに従って管理してください
- workspace フォルダ内の案件データは機密情報を含む場合があります。適切なアクセス制限を設定してください
