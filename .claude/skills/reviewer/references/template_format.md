# テンプレート記述例

ユーザーがテンプレート要件を入力する際の参考フォーマットです。

---

## 例1: ものづくり補助金テンプレート（簡易版）

```yaml
補助金名: ものづくり補助金（一般型）

文字数制限:
  - 事業概要: 500字以内
  - 技術的優位性: 2000字以内
  - 事業化実現性: 1500字以内
  - 実施体制: 500字以内

必須記載項目:
  - 現状の課題
  - 開発する技術の内容
  - 既存技術との比較
  - 売上計画（3年後まで）
  - 実施スケジュール
  - 実施体制（担当者・役割）

必須添付書類:
  - ガントチャート
  - 見積書
  - 決算書（直近3期）

審査項目:
  - 技術的優位性（40点）
  - 事業化実現性（30点）
  - 政策適合性（20点）
  - 費用対効果（10点）
```

## 例2: 事業再構築補助金テンプレート（簡易版）

```yaml
補助金名: 事業再構築補助金

文字数制限:
  - 事業の全体像: 1000字以内
  - 既存事業の課題: 1000字以内
  - 新規事業の内容: 2000字以内
  - 実施体制: 500字以内
  - 収支計画の根拠: 1500字以内

必須記載項目:
  - 既存事業の売上推移（コロナ前後比較）
  - 新規事業での製品・サービス内容
  - 市場調査結果
  - 売上計画（5年後まで）
  - 設備投資内容
  - 人員計画

必須添付書類:
  - 売上高推移グラフ
  - 事業計画書（5年分）
  - 認定支援機関の確認書
  - 見積書

審査項目:
  - 事業化実現性（30点）
  - 再構築の必要性（25点）
  - 市場の成長性（20点）
  - 実施体制（15点）
  - 政策適合性（10点）
```

## 例3: シンプルなテンプレート記述（口頭入力用）

ユーザーが詳細なテンプレートを持っていない場合、以下のように簡潔に指定してもらう：

```
補助金名: IT導入補助金

主な制限:
- 全体で10ページ以内
- 導入するITツールの説明: 2000字
- 導入効果の試算: 必須（数値根拠）

必須項目:
- 現状の業務課題
- ITツール選定理由
- 導入後の業務フロー
- 費用対効果（投資回収期間）
```

## テンプレート入力時のコツ

### ユーザーへのガイダンス

「テンプレート要件を教えてください」と聞かれたら、以下を案内：

1. **最低限必要な情報:**
   - 各セクションの文字数制限
   - 必須記載項目

2. **あると助かる情報:**
   - 審査項目と配点
   - 必須添付書類
   - ページ数制限

3. **情報がない場合:**
   - 「公募要領はありますか？」
   - 「過去の採択事例は見ましたか？」
   - → なければ一般的な審査基準でレビュー

---

## テンプレート要件の抽出（PDFから）

公募要領PDFがアップロードされた場合、以下を抽出：

1. **様式番号**: 「様式1」「別紙A」等の記載
2. **文字数**: 「○○字以内」「○○ページ以内」
3. **必須項目**: 「以下を必ず記載すること」の箇所
4. **審査基準**: 「審査項目」「配点」の記載
5. **添付書類**: 「提出書類一覧」

これらを構造化してテンプレート定義を作成する。

## スキルでの活用方法

SKILL.md内で、ユーザーからテンプレート要件を聞き取る際：

```markdown
以下の形式で教えてください（分かる範囲で結構です）：

**補助金名:**
**文字数制限:** （例: 事業概要 500字以内）
**必須項目:** （例: 売上計画、実施体制）
**審査項目:** （例: 技術性、事業性）
**添付書類:** （例: ガントチャート、見積書）

または、公募要領のPDFをアップロードしていただければ、
そこから要件を抽出します。
```

---

このテンプレートフォーマットにより、ユーザーは構造化された形で要件を伝えられ、
スキルは効率的にチェックを実行できます。
