```
- 技術的優位性: 2000字以内
- 事業化実現性: 1500字以内
- 売上計画: 必須（根拠含む）

Claude:

[ファイルを分析]

# ものづくり補助金 申請書レビュー結果

## 1. テンプレート整合性チェック

### ✓ 適合項目
- 事業概要: 480字/500字（適正）
- 実施体制: 記載あり

### ⚠ 要修正項目

1. **技術的優位性セクション: 2,340字/2,000字（340字超過）**
   削減が必要な箇所:
   - 既存技術の詳細説明（p.3, 2段落目）: 約150字
     → 要点のみに圧縮可能
   - 他社製品の仕様比較表の補足説明: 約120字
     → 表で自明な部分は削除可
   - 導入機器の一般的な説明: 約100字
     → カタログ参照に置き換え可

2. **売上計画の積算根拠: 不十分**
   - 「5年後に売上5億円」の記載はあるが、根拠が不明確
   - 提案: 以下いずれかの方法で根拠を明示
     
     【方法A: 市場規模ベース】
     - 対象市場規模: ○○億円（データソース明記）
     - 想定シェア: ○%
     - 計算式を明示
     
     【方法B: 顧客ベース】
     - 既存顧客への拡販: ○億円
     - 新規開拓（想定社数×単価）: ○億円
     - 合計の積み上げ

### ❌ 致命的不備
- 実施スケジュール（ガントチャート）が添付されていない
  → これは必須項目のため、未提出だと審査対象外

## 2. 審査項目別評価

### 技術的優位性（配点40点） 評価: B+

**強み:**
- 新加工技術の原理が図解されている（p.4）
- 特許出願中である点が明記
- 試作品での性能試験データあり

**弱み:**
- 既存技術（A社製品）との比較が抽象的
  → 改善案: 「加工時間50%短縮、精度2倍向上」など定量比較
  
- 技術的リスクへの言及なし
  → 改善案: 「技術課題と対策」セクションを追加
    例）「加工条件の最適化に3ヶ月を想定。ただし、類似技術での
        経験があるため、当社開発部門（5名）で対応可能」

### 事業化実現性（配点30点） 評価: C

**強み:**
- 既存顧客10社の引き合い実績あり
- 実施体制が具体的（担当者名・役割）

**弱み:**
- 市場分析が不足
  → 改善案: 「○○調査レポート（2024）によると、当該市場は
     年率7%成長」など、データソースを明記
  
- 売上計画の根拠が弱い（前述）
  
- 競合分析がない
  → 改善案: 主要競合3社との比較表を追加

### 政策適合性（配点20点） 評価: B

**強み:**
- 生産性向上への貢献が明記されている
- 地域雇用（新規採用2名）への言及あり

**弱み:**
- 補助金の政策目的への直接的な言及が少ない
  → 改善案: 冒頭に「本事業は、ものづくり補助金の目的である
    『革新的サービス開発・試作品開発・生産プロセスの改善』
     に該当し...」と明記

### 費用対効果（配点10点） 評価: B

**強み:**
- 設備投資の見積書が詳細
- 投資回収期間（3年）が明示

**弱み:**
- 積算根拠の一部が不明確（「その他経費500万円」）
  → 改善案: 内訳を明記

## 3. 改善提案

### 提案1: 文字数削減（技術的優位性セクション）

**Before（2,340字の一部抜粋）:**
```
当社が開発する新しい加工技術は、従来の切削加工技術と比較して
大幅な性能向上を実現するものです。既存の技術においては、加工
時間や加工精度の面で課題があり、多くの企業が改善を試みてきま
したが、根本的な解決には至っていませんでした。そこで当社は、
独自の○○方式を開発し、これらの課題を解決することに成功しま
した。具体的には、従来の方式では...（以下、既存技術の詳細な
説明が200字以上続く）
```

**After（圧縮版）:**
```
当社の新加工技術は、独自の○○方式により以下を実現します：

【性能比較】（従来技術との比較）
- 加工時間: 50%短縮（60分→30分）
- 加工精度: 2倍向上（±0.1mm→±0.05mm）
- コスト: 30%削減

既存技術の課題であった△△を、□□方式により解決しました
（試作機での検証済み、データ添付）。
```

**削減効果:** 約150字削減、かつ説得力向上

### 提案2: 売上計画の根拠強化

**Before:**
```
5年後に売上5億円を目指します。
```

**After:**
```
【売上計画の積算根拠】

**既存顧客ベース:**
- 既存顧客10社への拡販: 2億円
  （現在の取引額1社あたり平均1,000万円 → 2,000万円に拡大）
  
**新規開拓:**
- 展示会での引き合い企業: 50社
- 商談中: 10社
- 成約見込み（保守的に20%）: 2社/年 × 5年 = 10社
- 新規売上: 2億円（1社あたり2,000万円 × 10社）

**周辺商材:**
- メンテナンス・消耗品: 1億円
  （導入顧客20社 × 年間500万円）

**合計: 5億円**

※ 市場規模（500億円）に対する当社シェア1%に相当し、
  実現可能性は高いと判断
```

---

### 提案3: 表現の改善（審査員に響く書き方）

| Before（弱い） | After（強い） | 理由 |
|---|---|---|
| 市場ニーズがある | 展示会で50社から引き合い、うち10社と商談中 | 具体的な数値・状況 |
| 技術的に可能 | 類似技術での開発実績あり（製品B、納入30社） | 実績ベースの根拠 |
| 売上向上が期待 | 既存顧客A社での試験導入により、20%のコスト削減を実証済み | 定量的な効果 |
| 革新的な技術 | 特許出願中（出願番号○○）、従来比50%の時間短縮 | 客観的な根拠 |

## 4. 総合評価と推奨アクション

### 現在の採択可能性: B-（改善により採択の見込みあり）

### 強み（維持すべき点）
1. ✓ 技術的独自性が明確（特許出願中）
2. ✓ 試作品での検証データあり
3. ✓ 実施体制が具体的（担当者名・役割分担明記）
4. ✓ 既存顧客での引き合い実績

### 弱み（改善必須）
1. ⚠ **実施スケジュール未添付** - 優先度: ★★★（致命的）
2. ⚠ **文字数超過（340字）** - 優先度: ★★★（高）
3. ⚠ **売上計画の根拠不足** - 優先度: ★★☆（高）
4. ⚠ **市場分析の弱さ** - 優先度: ★☆☆（中）

### 推奨アクション（優先順位順）

**【今週中に対応】**
1. 実施スケジュール（ガントチャート）の作成・添付
   → これがないと審査対象外のリスク
   
2. 技術的優位性セクションの340字削減
   → 上記の改善案を参照

**【来週対応】**
3. 売上計画の積算根拠を追加
   → 上記の「顧客ベース積算」を参考に
   
4. 市場分析データの追加
   → ○○調査レポート等のデータを引用

**【時間があれば】**
5. 技術的リスクと対策のセクション追加
6. 競合分析表の追加
7. 全体的な表現の改善（上記の表現改善案を参照）

### 改善後の期待される評価
- 技術的優位性: B+ → A-（文字数適正化、リスク対策追加）
- 事業化実現性: C → B+（売上根拠、市場分析強化）
- 総合評価: B- → B+（採択可能性 中 → 高）

---
レビュー実施日: 2026-03-13
対象ドラフト: draft_application.pdf
```

### 会話フロー

```
コンサルタント:
明日提出なので、最終チェックをお願いします。
致命的なミスがないか確認してください。

Claude:
承知しました。提出前の最終チェックを実施します。

# 提出前チェックリスト

## ✓ クリティカルチェック（致命的不備の確認）

### 形式面
- [✓] 文字数制限を全て守っている
- [✓] 必須添付書類が全て揃っている
  - ガントチャート: ✓
  - 見積書: ✓
  - 決算書: ✓
- [✓] ページ番号あり
- [⚠] 誤字脱字: 2箇所発見（p.5「実施時期」→「実施体制」、p.8「競合他者」→「競合他社」）

### 内容面
- [✓] 全審査項目に回答
- [✓] 数値に根拠あり
- [✓] 図表に番号・キャプションあり
- [✓] 実施体制に担当者名あり

## ⚠ 要確認事項（重要度順）

1. **誤字の修正（2箇所）** - p.5, p.8
2. **図3の画質が低い** - 審査員が読めない可能性
3. **見積書の日付が古い** - 3ヶ月前（最新版推奨）

## ✓ 総合判定

**提出可能レベル: A-**

上記の誤字2箇所を修正すれば、問題なく提出できます。
見積書と図の画質は「できれば改善」レベルです。

最終確認事項:
- [ ] 誤字の修正
- [ ] PDF変換後の確認（レイアウト崩れなし）
- [ ] ファイル名が指定通りか
- [ ] 提出サイトの動作確認

お疲れ様でした。良い結果をお祈りしています！
```

## シナリオ3: 特定セクションの深掘りレビュー

### 会話フロー

```
コンサルタント:
前回のレビューで「技術的優位性が弱い」と言われました。
このセクションだけ、詳しく見てください。

Claude:
承知しました。技術的優位性セクションを詳細分析します。

# 技術的優位性セクション 詳細レビュー

## 現状分析

### 記載内容（要約）
- 新しい○○加工技術を開発
- 従来技術より「高速・高精度」
- 特許出願中

### 弱点

1. **比較が抽象的**
   - 「高速」「高精度」だけでは審査員は評価不可
   - 他社製品名の記載なし
   
2. **定量データが少ない**
   - 数値比較が1つだけ
   - 試験データの詳細なし
   
3. **技術的実現可能性の根拠が薄い**
   - 「開発中」とあるが、どこまで進んでいるか不明
   - リスクへの言及なし

## 改善提案

### 提案1: 他社製品との明確な比較表

**追加すべき内容:**

| 項目 | 当社技術 | A社製品 | B社製品 | 従来技術 |
|---|---|---|---|---|
| 加工時間 | 30分 | 45分 | 60分 | 60分 |
| 加工精度 | ±0.05mm | ±0.08mm | ±0.1mm | ±0.1mm |
| 設備価格 | 3,000万円 | 5,000万円 | 4,500万円 | 2,000万円 |
| ランニングコスト/年 | 200万円 | 350万円 | 300万円 | 400万円 |

表の後に:
「A社製品と比較して、加工時間33%短縮、精度37%向上を実現。
 設備価格は高いが、ランニングコストが低く、3年で投資回収可能。」

### 提案2: 開発進捗の明示

**現在の記載:**
「現在開発中です。」

**改善案:**
「開発進捗: 80%完了
- 基本原理の検証: 完了（2024年6月）
- 試作機による性能試験: 完了（2024年9月）
- 実機での実証試験: 実施中（2025年3月完了予定）

残課題: 加工条件の最適化（3ヶ月で完了見込み）
対策: 当社開発部門5名を専任配置、外部専門家1名を技術顧問として招聘」

### 提案3: 技術的優位性を支えるエビデンスの追加

**追加すべき資料:**
1. 試作機での性能試験結果（グラフ・表）
2. 特許出願内容の概要（出願番号明記）
3. 外部機関による評価（あれば）
4. 類似技術での実績（当社の過去の開発案件）

### Before/After 全体構成

**Before:**
```
【技術的優位性】（現在: 約800字）

当社が開発する新しい○○加工技術は、従来技術と比較して
高速・高精度を実現します。特許出願中です。現在開発中で
あり、今後実用化を目指します...（以下抽象的な説明）
```

**After:**
```
【技術的優位性】（改善後: 約1,800字）

## 1. 技術の概要
当社の○○加工技術は、独自の△△方式により、従来技術の
課題であった加工時間と精度を大幅に改善します。

## 2. 他社製品との比較
（上記の比較表を挿入）

当社技術の優位性:
- 加工時間: A社比33%短縮
- 加工精度: 業界トップレベル（±0.05mm）
- 総コスト: 3年で投資回収可能

## 3. 技術的実現可能性
開発進捗: 80%完了
（進捗詳細を記載）

類似技術での実績:
- 製品B（2020年開発）: 納入実績30社
- 製品C（2022年開発）: 業界シェア15%獲得

## 4. 知的財産権
特許出願中（出願番号: 特願2024-XXXXXX）
発明の名称: ○○加工方法および装置

## 5. 技術課題と対策
残課題: 加工条件の最適化
対策: 専任チーム5名 + 外部専門家1名
期間: 3ヶ月（余裕をもったスケジュール）
```

## 推奨アクション

1. **比較表の作成**（優先度: 高）
   → Excel等で作成し、画像として挿入
   
2. **開発進捗の具体化**（優先度: 高）
   → タイムライン図を追加
   
3. **試験データの添付**（優先度: 中）
   → グラフ・表で可視化
   
4. **文字数調整**（優先度: 中）
   → 他のセクションを圧縮して、技術セクションに配分

この改善により、技術的優位性の評価を B → A に引き上げることが
期待できます。
```
