# 補助金申請書レビュー・改善スキル

## 概要

補助金申請支援コンサルタント向けの専門スキルです。
クライアントの申請書ドラフトを審査員の視点でレビューし、採択確度を高めるための具体的な改善提案を行います。

## 主な機能

✅ **審査基準に基づくレビュー** - 外部審査員の評価視点で申請書を分析  
✅ **テンプレート整合性チェック** - 文字数制限・必須項目の網羅性確認  
✅ **具体的改善提案** - Before/After形式で文字数削減案、根拠強化、表現改善  
✅ **採択可能性の評価** - 現状の強み・弱みの明確化と優先順位付き改善案  

## 対応フォーマット

- PDF（申請書、公募要領、参考資料）
- Word（.docx）
- Excel（.xlsx）売上計画表など
- テキスト形式の申請書

## 使用方法

### 基本的な使い方

1. **申請書ドラフトをアップロード**
2. **補助金のテンプレート要件を伝える**（文字数制限、必須項目等）
3. **レビュー結果を受け取る**

### 使用例1: 全体レビュー

```
ユーザー: ものづくり補助金の申請書をレビューしてください。
         テンプレート要件は以下です：
         - 事業概要: 500字以内
         - 技術的優位性: 2000字以内
         - 事業化実現性: 1500字以内
         - 売上計画: 必須（根拠含む）

Claude: [申請書を読み込み、以下を実施]
        1. テンプレート整合性チェック
        2. 審査項目別評価
        3. 改善提案（Before/After）
        4. 総合評価と優先順位
```

### 使用例2: テンプレート不明な場合

```
ユーザー: この申請書、全体的にどうですか？

Claude: 承知しました。まず確認させてください：
        - 対象の補助金名は何ですか？
        - テンプレートの文字数制限はありますか？
        
        （要件不明の場合は一般的な審査基準でレビュー）
```

### 使用例3: 特定セクションのレビュー

```
ユーザー: 技術的優位性のセクションだけ詳しく見てください。
         他社との差別化が弱いと言われました。

Claude: [該当セクションに集中してレビュー]
        - 現状の弱点分析
        - 他社比較の具体的な書き方提案
        - Before/After形式で改善案
```

## スキルの強み

### 1. 審査員視点の徹底
単なる文章チェックではなく、実際の審査員（中小企業診断士、大学教授等）が
どう評価するかを意識したレビュー

### 2. 具体的な改善提案
「もっと具体的に」ではなく、Before/After形式で実際の改善文例を提示

### 3. 優先順位の明確化
「あれもこれも直して」ではなく、致命的な不備から順に優先順位を提示

### 4. 多様な補助金に対応
ものづくり、事業再構築、IT導入など、幅広い補助金に対応可能

## 開発背景

補助金申請書の審査では、以下の課題がありました：

1. **文字数超過による足切り** - 1文字でも超過すると審査対象外
2. **必須項目の漏れ** - チェックリスト的な確認が必要
3. **根拠のない数値計画** - 審査員が最も減点する箇所
4. **審査員視点の欠如** - 申請者目線と審査員目線のギャップ

このスキルは、これらの課題を体系的に解決します。
