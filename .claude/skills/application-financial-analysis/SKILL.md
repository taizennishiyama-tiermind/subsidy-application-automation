# 財務分析スキル

`/analyze` 実行時の標準スキル。外部から取り込んだ分析 skill を、このリポジトリの `workspace/{案件名}` フローに合わせて調整している。

## 使い方

1. まず `workspace/{案件名}/hearing_notes.md` を読む
2. 決算書 PDF / Excel / 試算表 / 概算値のどれで進めるかを明示する
3. 使った資料、対象期間、概算の有無を冒頭に整理する
4. 必要に応じて以下を使う
   - `scripts/calculate_ratios.py`: 指標計算
   - `scripts/generate_excel.py`: Excel レポート生成
   - `scripts/generate_word_report.js`: Word レポート生成
   - `references/financial_indicators.md`: 指標定義確認
   - `references/industry_benchmarks.json`: 業界平均確認
5. 結果は必ず `workspace/{案件名}/company_data.md` に集約する

## 入力方針

### 優先順位

1. 直近3期の決算書・試算表
2. 法人事業概況説明書 / 確定申告書
3. 月次資料、借入一覧、資金繰り表
4. ヒアリングによる概算値

### 形式別ルール

- PDF / 画像: 数値抽出後に主要数値を再確認する
- Excel / CSV: シート構造を見て勘定科目を対応付ける
- 手入力 / 概算: 概算値であることを明記する

## 主要な分析観点

### 1. 申請要件に直結する指標

- 付加価値額 = 営業利益 + 人件費 + 減価償却費
- 給与支給総額
- 労働生産性
- 最低賃金水準

### 2. 財務健全性の指標

- 営業利益率
- ROA / ROE
- 自己資本比率
- 流動比率
- 固定比率
- 債務償還年数
- インタレストカバレッジレシオ

### 3. 申請書に転用する視点

- 何が強みか
- どこが弱いか
- 補助事業でどの数値がどう改善するか
- 補助金がなくても事業継続性があるか

## 補助金別の重点

- ものづくり補助金: 付加価値額、給与支給総額、最低賃金、設備投資効果
- IT導入補助金: 労働生産性、業務時間削減、ミス削減、収益改善
- 新事業進出系: 既存事業との整合、新規事業の採算性、返済余力
- 持続化補助金: 課題の根拠、投資対効果、販路拡大の妥当性

## 実務フロー

### Phase 1: データ整理

- 3期比較できる形にそろえる
- 不足科目があれば不足として明記する
- 人件費や減価償却費が取れない場合は、付加価値額の精度に注意を入れる

### Phase 2: 指標計算

- 手計算でもよいが、再利用可能な場合は `scripts/calculate_ratios.py` を優先する
- 業界平均比較が必要なら `references/industry_benchmarks.json` を参照する

### Phase 3: 解釈

- 良好 / 普通 / 要改善の評価をつける
- 評価理由を申請書で使える文に落とす
- 弱い指標は「補助事業で改善する筋道」まで書く

### Phase 4: 出力

最低でも `company_data.md` に以下を残す:

- 使用資料
- 3期比較の主要数値
- 付加価値額と給与支給総額の推移
- 主要指標の評価
- SWOT
- 将来計画のたたき台
- 未確定数値と再確認項目

必要に応じて補助資料として以下も生成する:

- `workspace/{案件名}/financial_ratios.json`
- `workspace/{案件名}/財務分析_*.xlsx`
- `workspace/{案件名}/財務分析レポート_*.docx`

## company_data.md の推奨構成

### 1. 入力条件

- 使用資料
- 対象年度
- 概算値の有無
- AI がどの方式で読んだか

### 2. 企業概要

- 会社名
- 代表者
- 所在地
- 業種
- 主な事業
- 従業員数

### 3. 財務サマリー

- 売上高、営業利益、経常利益の3期比較
- 付加価値額の3期比較
- 給与支給総額の3期比較

### 4. 指標評価

- 収益性
- 安全性
- 成長性
- 返済能力
- 人件費 / 労働生産性

### 5. SWOT

- 強み
- 弱み
- 機会
- 脅威

### 6. 数値計画の論点

- どの売上増加要因を積み上げるか
- どのコスト削減を見込むか
- 目標値が現実的か

## 注意

- 概算値は概算と明記する
- 付加価値額に人件費を入れ忘れない
- 根拠のない右肩上がり計画を書かない
- 指標の良し悪しだけで終わらず、申請書でどう使うかまで落とす
- 出力ファイルを作っても、最終的な正本は `company_data.md` を基準にする
