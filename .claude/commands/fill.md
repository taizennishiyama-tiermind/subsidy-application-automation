# テンプレートへの書き込み実行

申請書データをExcel/Wordテンプレートに自動で書き込みます。
**書き込み前に、登録済みテンプレートの更新日・公募回を確認し、最新版かどうかをユーザーに確認します。**

## 引数

$ARGUMENTS = 案件名（省略時はユーザーに確認する）

## 前提条件

- `workspace/{案件名}/draft_application.md` が作成済みであること
- `workspace/{案件名}/application_data.json` が存在すること（なければ Step 2 で生成）
- 対象補助金のテンプレートが `grants/{補助金名}/templates/` に配置済みであること
- セルマッピングが `/add-template` で設定済みであること
- `TEMPLATE_SKILL.md` または同等のテンプレート解析結果が存在すること

## 手順

### Step 1: 対象補助金・テンプレートの特定

1. `workspace/{案件名}/strategy.md` から対象補助金名を取得する
2. `grants/{補助金名}/templates/` 内のファイル一覧を確認する
3. `grants/{補助金名}/templates/` 配下のテンプレート実体と `TEMPLATE_SKILL.md` を確認する

### Step 2: テンプレートの更新状況を確認・ユーザーに報告

保存済みのテンプレートフォルダと `TEMPLATE_SKILL.md` の内容をもとに、以下のフォーマットでユーザーに報告する:

```
【使用予定のテンプレート確認】

補助金名  : {補助金名}
ファイル名 : {テンプレートファイル名}
公募回・年度: {fiscal_year}
登録日    : {registered_at}
取得元    : {source}
バージョンメモ: {version_note}（なければ「なし」）

このテンプレートで進めますか？
最新の公募要領・テンプレートを確認の上、ご判断ください。
（更新が必要な場合は「更新する」と入力してください）
```

- **「はい」「OK」等** → Step 3 へ進む
- **「更新する」** → 「`/add-template {補助金名}` を実行して新しいテンプレートを登録してください」と案内して終了
- テンプレート本体または `TEMPLATE_SKILL.md` が存在しない場合 → 「テンプレートの解析情報が見つかりません。`/add-template {補助金名}` を先に実行してください」と案内して終了

### Step 3: application_data.json の確認・生成

`workspace/{案件名}/application_data.json` を読み込む（存在する場合）。
存在しない場合は、`workspace/{案件名}/draft_application.md` と `workspace/{案件名}/company_data.md` から、
テンプレートの各フィールドに対応するデータをJSON形式で抽出・整形する。

```json
{
  "company_name": "株式会社XXX",
  "representative": "山田太郎",
  "business_overview": "...",
  "challenge_1": "...",
  "sales_year0": "85000",
  "sales_year1": "90000"
}
```

JSONのキーは `grants/{補助金名}/scripts/fill_template.py` のセルマッピングに対応させる。

### Step 4: テンプレート解析結果との整合確認

1. `grants/{補助金名}/templates/TEMPLATE_SKILL.md` を読み込む
2. Claude Skills により整理済みの全シート一覧、変数一覧、セル対応、記載上の注意点を確認する
3. `application_data.json` のキーとテンプレート変数の差分を確認する
4. **全シートを再確認し、入力対象、参照専用、自動計算、説明用の分類に漏れがないか確認する**
5. プレースホルダーや Named Range が無い場合でも停止しない。`subsidy-application-filler` スキルを使い、ラベル、結合セル、表構造、見出し、欄サイズ、周辺文言から**今回のテンプレート版の実際の入力位置を再解決**し、マッピング定義を作成する
6. Word が参考様式や記載項目一覧で直接差し込みできない場合は、設問順に対応する完成版本文を生成し、必要なら新しい `.docx` または `.md` を完成版として出力する
7. Excel が複雑表の場合は、結合セルの左上、数式セル保護、行間隔、動的表を分析し、必要に応じてマッピングを補正する
8. **申請に提出するシートの入力箇所は全て特定し、埋められる情報は全て `application_data.json` に反映する**
9. **「記入例」「記入方法」「説明」シートには書き込まない。参照して得たルールは提出対象シートへの入力にのみ反映する**
10. **提出対象シートの入力欄に例文・記入見本・ダミーテキストが入っている場合は、実値を書き込む前に必ず削除する**
11. 足りない項目がある場合は `/draft {案件名}` に戻って補完するか、ユーザーに不足を確認する
12. **必須項目の差し込み先を解決できない場合は、空欄のまま出力せずエラーとして扱う**
13. 問題なければ Step 5 へ進む

### Step 5: スクリプト実行

テンプレートの形式（Excel / Word）を確認してから実行する。

**Excel（.xlsx）の場合:**
```bash
.venv/bin/python "grants/{補助金名}/scripts/fill_template.py" fill \
  --template "grants/{補助金名}/templates/{テンプレートファイル.xlsx}" \
  --data "workspace/{案件名}/application_data.json" \
  --output "workspace/{案件名}/申請書_{補助金名}.xlsx"
```

**Word（.docx）の場合:**
```bash
.venv/bin/python "grants/{補助金名}/scripts/fill_template.py" fill \
  --template "grants/{補助金名}/templates/{テンプレートファイル.docx}" \
  --data "workspace/{案件名}/application_data.json" \
  --output "workspace/{案件名}/申請書_{補助金名}.docx"
```

複数のテンプレートファイルがある場合（Excel + Word の両方等）は、それぞれに対して実行する。

**プレースホルダー未設定テンプレートの場合の追加ルール:**

- `subsidy-application-filler` スキルの分析スクリプトでテンプレート構造を確認する
- `workspace/{案件名}/mappings/` にテンプレートごとのマッピング JSON を生成・保存する
- **全シートを見て特定した提出対象シートの入力項目を、テンプレートごとに漏れなくマッピングする**
- **記入例や記入方法シートはマッピング対象に含めない**
- **固定セルは初期候補として扱い、毎回ラベルアンカーと周辺構造で差し込み先を再確認する**
- **必須項目は「未解決ゼロ」を確認してから出力する**
- **提出対象シートの入力欄に既存の例文やダミー文が入っている場合は、書き込み前に削除してから実値を入れる**
- 元テンプレートへ直接差し込み可能なら差し込む
- 元テンプレートへ直接差し込みできない場合でも、完成版相当の `.docx`、`.xlsx`、または `.md` を生成し、提出・転記作業が止まらない状態にする
- 推定で埋めた箇所は、どの根拠でその位置に入れたかを `TEMPLATE_SKILL.md` または案件メモに残す

### Step 6: 完了確認

1. 出力ファイルのパスをユーザーに伝える
2. 書き込まれた項目数と、確認したシート数を報告する
3. 内容の確認を依頼する
4. 完成版ファイルと、今回有効だったテンプレート運用メモを次回再利用できるよう整理する
5. プレースホルダー無しテンプレートに対して推定マッピングを行った場合は、再利用用のマッピングファイルと注意点を必ず保存する
6. **提出対象シートの必須欄が全て埋まっていることを最終確認する**

### Step 7: 情報蓄積

今回の出力で得た情報は、性質に応じて蓄積先を分ける:

- `grants/{補助金名}/` に蓄積するもの
  どのテンプレート版を使ったか、手修正が必要だった箇所、次回同じ補助金で注意すべき点
- `know-how/` に蓄積するもの
  どの変数が不足しやすかったか、社内運用上の確認ポイント

### Step 8: ノウハウ蓄積の確認

完了前に、必ずユーザーへ次を確認する:

```text
今回の工程で得た補助金情報、テンプレート情報、運用ノウハウを次回以降のために蓄積しますか？
（はい / いいえ）
```

**ユーザーが `はい` と答えた場合:**

1. 補助金が未登録なら `grants/{補助金名}/` を作成し、必要なら `/add-grant {補助金名}` 相当の処理を行う
2. 今回使った制度情報、公募回差分、注意点を `grants/{補助金名}/` に反映する
3. 取得済み PDF を `grants/{補助金名}/docs/` に自動格納する
4. 使用したテンプレートファイルを `grants/{補助金名}/templates/` に自動格納する
5. `TEMPLATE_SKILL.md`、マッピング根拠、運用メモを更新する
6. 自社独自の運用ノウハウを `know-how/` に整理して保存する

**ユーザーが `いいえ` と答えた場合:**

- 今回の案件成果物は `workspace/{案件名}/` にのみ残し、`grants/` と `know-how/` への反映は行わない

## 完了メッセージ

「テンプレートへの書き込みが完了しました。

出力ファイル: `workspace/{案件名}/申請書_{補助金名}.{xlsx または docx}`
使用テンプレート: {ファイル名}（{fiscal_year}版 / 登録日: {registered_at}）

内容を確認し、必要に応じて手動で微調整してください。
次の公募回が開始された際は `/add-template {補助金名}` でテンプレートを更新してください。

今回の工程で得た補助金情報、テンプレート情報、運用ノウハウを次回以降のために蓄積しますか？
（はい / いいえ）」 

## エラー時

- テンプレートが見つからない → `grants/{補助金名}/templates/` への配置と `/add-template` の実施を案内
- TEMPLATE_SKILL.md が存在しない → `/add-template {補助金名}` の実施を案内
- セルマッピングが未設定 → `/add-template {補助金名}` の実施を案内
- Python環境がない → `bash scripts/setup.sh` の実行を案内
- データが不足 → 不足項目を表示し `/draft` での追記を案内
