# ヒアリング実施・再開

既存案件のヒアリング（Phase 1）を実施または再開します。

## 引数

$ARGUMENTS = 案件名（省略時はユーザーに確認する）

## 前提条件

- `workspace/{案件名}/` が存在すること（`/new-case` で作成済み）

## 手順

### Step 1: 既存記録の確認

1. `workspace/{案件名}/hearing_notes.md` を読み込む
2. 既にヒアリング済みの項目と、未完了の項目を特定する

### Step 2: ヒアリング再開

1. `.claude/skills/hearing-expert/SKILL.md` を読み込む
2. 未完了の項目から質問を再開する
3. **1問ずつ質問する**（一度に大量の質問を投げない）
4. 回答に応じて深掘りする
5. 結果を `workspace/{案件名}/hearing_notes.md` に随時記録する

### Step 3: ヒアリング完了判定

以下が全て揃ったら Phase 1 完了:
- [ ] 基本情報（会社名、業種、従業員数、年商）
- [ ] 事業の強み（3つ以上）
- [ ] 現状の課題（具体的に）
- [ ] 投資計画（何を、いくらで、なぜ）
- [ ] 将来ビジョン
- [ ] 補助金関連の確認事項（過去の申請歴、決算書の準備状況等）

## 完了メッセージ

「ヒアリングが完了しました。次のステップ:
- `/analyze {案件名}` → 決算書等の財務データを分析
- `/match {案件名}` → 最適な補助金を提案（財務データなしでも可）」

## エラー時

- 案件フォルダが存在しない → `/new-case` の実施を案内
- hearing_notes.md が空 → 最初からヒアリングを開始
