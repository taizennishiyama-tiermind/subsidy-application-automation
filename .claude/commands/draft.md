# 申請書ドラフト作成（Phase 4）

ヒアリング結果と財務分析をもとに、採択率の高い申請書ドラフトを作成します。

## 引数

$ARGUMENTS = 案件名（省略時はユーザーに確認する）

## 前提条件

- `workspace/{案件名}/hearing_notes.md` が完成していること
- `workspace/{案件名}/strategy.md` に補助金が選定済みであること

## 手順

### Step 1: ストーリー設計

1. `workspace/{案件名}/` の全ファイルを読み込む
2. 選定した補助金の `grants/{補助金名}/overview.md` を読み込む
3. `.claude/skills/storytelling/SKILL.md` を参照し、最適なストーリーパターンを選択する
4. `.claude/skills/review-criteria/SKILL.md` の採択事例も参考にする
5. `ノウハウ/` フォルダにファイルがあれば読み込み、自社独自の知見を反映する
6. ストーリー構成案を策定し、`workspace/{案件名}/strategy.md` に追記する
6. **ユーザーに構成案を確認してもらう**（承認後に次のStepへ）

### Step 2: 申請書本文の作成

1. ストーリー構成に基づき、申請書の各セクションを順番に作成する
2. 各セクション完成時にユーザーに逐次確認する
3. 品質基準を満たすよう以下を確認:
   - 審査基準の全項目をカバーしているか
   - 全ての数値に根拠があるか
   - 図表（before/after表）の配置案を含めているか
   - ページ数制限を意識しているか
4. ドラフトを `workspace/{案件名}/draft_application.md` に出力する

### Step 3: テンプレート書き込み準備

1. `grants/{補助金名}/templates/` にテンプレートファイルがあるか確認する
2. **テンプレートがある場合:**
   - 申請書データをJSON形式で `workspace/{案件名}/application_data.json` に出力
   - JSON構造は `grants/{補助金名}/scripts/fill_template.py` のセルマッピングに対応させる
3. **テンプレートがない場合:**
   - markdownドラフトのみ出力
   - 「テンプレートを配置して `/fill {案件名}` でExcel/Wordに書き込めます」と案内

## ストーリー構成の基本

```
現状（自社の強み・実績）→ 課題（環境変化・ニーズ）→ 解決策（補助事業）→ 効果（数値目標）
```

## 完了メッセージ

「申請書ドラフトが完成しました。次のステップ:
- `/review {案件名}` → 審査基準に照らしたセルフレビュー
- `/fill {案件名}` → Excel/Wordテンプレートへの書き込み（テンプレート配置済みの場合）」

## エラー時

- strategy.md に補助金が未選定 → `/match` の実施を案内
- ヒアリング不足 → 不足項目を表示し `/hearing` で補完を案内
