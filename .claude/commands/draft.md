# 申請書ドラフト作成（Phase 4）

ヒアリング結果と財務分析をもとに、採択率の高い申請書ドラフトを作成します。

## 引数

$ARGUMENTS = 案件名（省略時はユーザーに確認する）

## 前提条件

- `workspace/{案件名}/hearing_notes.md` が完成していること
- `workspace/{案件名}/strategy.md` に補助金が選定済みであること
- **`workspace/{案件名}/templates/{補助金名}/` にテンプレート実体があり、再解析可能であること（必須）**

## 手順

### Step 0: テンプレートの確認（必須チェック）

**申請書作成前に、公式テンプレートの登録を必ず確認する。**

1. `workspace/{案件名}/templates/{補助金名}/` 配下に解析対象のテンプレート実体があるか確認する

**テンプレートが登録済みの場合:**

```
✅ テンプレートが登録済みです

  ファイル名 : {ファイル名}
  公募回・年度: {年度}
  登録日    : {登録日}

このテンプレートを使って申請書データを作成します。
（最新版かどうか念のご確認をお願いします）
```

さらに以下を実施してから Step 1 へ進む。

- `TEMPLATE_SKILL.md` を読み込み、設問構造・変数一覧・注意点を確認する
- ドラフト本文をテンプレート前提の粒度で作る方針を明示する
- completion 時に `/fill` まで見据えて `application_data.json` を作ることを案内する

**テンプレートが未登録の場合（進行を保留）:**

```
⛔ テンプレートが未登録のため、ドラフト作成を開始できません。

テンプレートは申請書の精度と自動書き込みに必要な必須資料です。

【次のアクション】
補助金の公式テンプレートファイル（.xlsx / .docx）を
`workspace/{案件名}/templates/{補助金名}/` に配置してください。

配置後に `/add-template {補助金名}` を実行すると、
自動的に整理・解析し、情報が揃い次第ドラフト作成を再開します。

テンプレートの入手先:
→ grants/{補助金名}/templates/README.md に公式URLが記載されています
```

→ テンプレートが配置されたら `/add-template {補助金名}` を実行し、
   完了後にドラフト作成を再開する。

### Step 1: 参照資料の読み込み

1. `workspace/{案件名}/` の全ファイルを読み込む
2. 選定した補助金の `grants/{補助金名}/overview.md` を読み込む
3. `grants/{補助金名}/docs/` フォルダを確認し、PDFがあれば内容を読み込む
   - `docs/` の内容と `overview.md` に差異がある場合は `docs/` を優先する
4. `.claude/skills/grant-application-writer/SKILL.md` を参照し最適なストーリーパターンを選択する
5. `.claude/skills/reviewer/SKILL.md` の評価観点も参考にする
6. `know-how/` フォルダにファイルがあれば読み込み、知見を反映する
7. `workspace/{案件名}/templates/{補助金名}/` 配下の `TEMPLATE_SKILL.md` があれば読み込み、テンプレートの記載要件を把握する
8. 同じ補助金の過去案件由来のノウハウが `know-how/` にあれば優先参照し、使う観点を整理する

### Step 2: ストーリー構成案の確認

1. ストーリー構成案を策定し、ユーザーに提示する
2. **ユーザーの承認を得てから Step 3 へ進む**（変更要望があれば対応）

### Step 3: 申請書本文の作成

1. 承認されたストーリー構成に基づき申請書の各セクションを作成する
2. 品質基準を満たすよう以下を確認:
   - 審査基準の全項目をカバーしているか
   - 全ての数値にヒアリング記録・財務データからの根拠があるか
   - Before/After 比較表が配置されているか
   - ページ数制限を意識しているか（公募要領に記載あれば厳守）
   - テンプレートがある場合は、各欄の設問意図と入力欄の粒度に沿って文章量を調整しているか
3. ドラフトを `workspace/{案件名}/draft_application.md` に出力する

### Step 4: テンプレート書き込みデータの生成

`workspace/{案件名}/application_data.json` を生成する:

1. `workspace/{案件名}/templates/{補助金名}/` 配下の `TEMPLATE_SKILL.md` があれば、そこに定義された変数リストに基づいてJSONを構築する
2. なければ `grants/{補助金名}/scripts/fill_template.py` のセルマッピングからキーを参照する
3. ドラフト内容をJSON形式に変換して保存する
4. テンプレート登録済みの場合は、**どの変数をどの欄に入れるかの対応関係を確認したうえで** JSON を生成する
5. テンプレートが複数ある場合は、テンプレートごとの差分も整理する

テンプレートが登録済みで `/fill {案件名}` を実行できる場合:
- 「テンプレートへの書き込み準備が整いました。`/fill {案件名}` で出力できます」と案内する

### Step 5: 情報蓄積

案件で得た情報は、性質に応じて蓄積先を分ける:

- `grants/{補助金名}/` に蓄積するもの
  補助金ごとの訴求ポイント、公募回固有の注意点、テンプレート固有の注意点
- `know-how/` に蓄積するもの
  次回も使える表現、自社として通りやすかった構成、チェックポイント

## ストーリー構成の基本

```
現状（自社の強み・実績）→ 課題（環境変化・ニーズ）→ 解決策（補助事業）→ 効果（数値目標）
```

## 完了メッセージ

「申請書ドラフトが完成しました。

次のステップ:
- `/review {案件名}` → 審査基準に照らしたセルフレビュー（推奨）
- `/fill {案件名}` → Excel/Wordテンプレートへの書き込み

（テンプレート未登録の場合）
💡 公式テンプレートを入手したら `workspace/{案件名}/templates/{補助金名}/` に配置後、
`/add-template {補助金名}` → `/fill {案件名}` で申請書ファイルを自動生成できます。」

## エラー時

- strategy.md に補助金が未選定 → `/match {案件名}` の実施を案内
- ヒアリング不足 → 不足項目を表示し `/hearing {案件名}` で補完を案内
