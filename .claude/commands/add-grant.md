# 新規補助金の追加

新しい補助金の種類をこの環境に追加します。
**補助金情報は蓄積型の資産です。** 追加された補助金は次回以降の全案件で活用されます。

## 引数

$ARGUMENTS = 補助金名（省略時はユーザーに確認する）

## 実行されるタイミング

1. ユーザーが明示的に `/add-grant {補助金名}` を実行したとき
2. `/match` でWeb検索により新しい有力補助金が見つかり、ユーザーが登録を承認したとき

## 手順

### Step 1: 重複チェック

`grants/` フォルダに同名のフォルダが既に存在するか確認する。
- 存在する → 「既に登録されています。上書きしますか？」と確認
- 存在しない → Step 2 へ進む

### Step 2: フォルダ構造の作成

以下のディレクトリとファイルを作成する:

```
grants/{補助金名}/
├── overview.md                    ← 制度概要・申請要件・審査基準
├── docs/
│   └── README.md                  ← 公募要領・交付規定等のPDF格納手順
├── templates/
│   └── README.md                  ← テンプレートファイルの格納手順
└── scripts/
    └── fill_template.py           ← テンプレート書き込みスクリプト
```

### Step 3: overview.md の作成

**情報の優先順位（ユーザーへの質問を最小化する）:**
1. `/match` のWeb検索結果で取得済みの情報があれば、それをそのまま使用する
2. 不足している場合のみWebで追加調査する
3. Webでも見つからない場合のみユーザーに公募要領の提供を依頼する

**収集する情報:**
- 正式名称・所管省庁・公式サイトURL
- 制度概要（200字程度）
- 対象者要件（業種・規模・地域等）
- 補助上限額・補助率
- 対象経費
- 申請書の主要記載項目
- 審査基準・配点
- 最新の公募スケジュール（申請締切日）
- 採択のポイント
- よくある不採択理由

既存の `grants/*/overview.md` と同じ形式で作成する。

### Step 4: docs/README.md の作成

格納すべき公式資料（公募要領・交付規定・Q&A等）の種類とダウンロード元URLを記載する。
URLが判明している場合は直接記載する（ユーザーがすぐダウンロードできるように）。
既存の `grants/*/docs/README.md` と同じ形式で作成する。

### Step 5: templates/README.md の作成

格納すべきテンプレートファイルの一覧とダウンロード元URLを記載する。
URLが判明している場合は直接記載する。

### Step 6: scripts/fill_template.py の作成

既存のスクリプト（例: `grants/IT導入補助金/scripts/fill_template.py`）をベースに作成する。

- `EXCEL_CELL_MAPPING` と `WORD_PLACEHOLDER_MAPPING` はコメントアウトで用意
- テンプレートファイルがアップロードされた後に `/add-template` で実際のマッピングを設定

### Step 7: CLAUDE.md の更新

CLAUDE.md の「保存済み補助金（grants/ フォルダ）」テーブルに新しい補助金を追加する。

### Step 8: 蓄積サマリーの表示

追加完了後、以下の形式でサマリーを表示する:

```
✅ {補助金名} を grants/ に蓄積しました

【登録された情報】
- 制度概要: grants/{補助金名}/overview.md
- 公式資料格納先: grants/{補助金名}/docs/
- テンプレート入手先メモ: grants/{補助金名}/templates/

【次のステップ（任意・後でもOK）】
1. 公募要領PDFを grants/{補助金名}/docs/ に配置する
   → 申請書作成時の精度が向上します

2. テンプレートファイルを `workspace/{案件名}/templates/{補助金名}/` に配置後、
   /add-template {補助金名} を実行する
   → 申請書を自動でExcel/Wordに出力できます

💡 次回から /match で自動的に候補として評価されます
```

## エラー時

- 同名のフォルダが既に存在する → 上書きするか別名にするか確認
- Web調査で情報が見つからない → ユーザーに公募要領の提供を依頼
