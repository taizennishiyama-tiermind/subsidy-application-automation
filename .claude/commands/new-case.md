# 新規案件開始 & ヒアリング開始

新しい補助金申請支援案件を開始し、Phase 1 ヒアリングを実施します。

## 引数

$ARGUMENTS は不要（対話で案件名を取得）

## 前提条件

- なし（最初のステップ）

## 手順

### Step 1: 案件情報の取得

ユーザーに以下を確認する:
1. 案件名（会社名 or 略称）
2. 今日の日付を取得する

### Step 2: ワークスペース作成

```bash
mkdir -p "workspace/{案件名}"
```

以下のファイルを新規作成する（中身は案件名と日付で初期化）:

- `workspace/{案件名}/hearing_notes.md` — ヒアリング記録
- `workspace/{案件名}/company_data.md` — 会社・財務データ
- `workspace/{案件名}/strategy.md` — 補助金選定・戦略
- `workspace/{案件名}/draft_application.md` — 申請書ドラフト

各ファイルはヘッダー（`# {案件名} - {ファイル種別}`、作成日、ステータス）のみで初期化する。

### Step 3: 対応補助金の確認

`grants/` フォルダ内の補助金フォルダ一覧を確認し、ヒアリング時の参考にする。

### Step 4: ヒアリング開始

`.claude/skills/hearing-expert/SKILL.md` を読み込み、その手法に従って Phase 1 を開始する。

**重要ルール:**
- **1問ずつ質問する**（一度に大量の質問を投げない）
- 回答に応じて深掘りする
- 結果は `workspace/{案件名}/hearing_notes.md` に随時記録する
- 対面ヒアリングの場合、お客様に寄り添った平易な言葉で質問する
- どの補助金に申請するかの目星をつけながらヒアリングする

### Step 5: ヒアリング完了判定

以下が全て揃ったら Phase 1 完了:
- [ ] 基本情報（会社名、業種、従業員数、年商）
- [ ] 事業の強み（3つ以上）
- [ ] 現状の課題（具体的に）
- [ ] 投資計画（何を、いくらで、なぜ）
- [ ] 将来ビジョン
- [ ] 補助金関連の確認事項（過去の申請歴、決算書の準備状況等）

## 完了メッセージ

「ヒアリングが完了しました。次のステップ:
- `/analyze {案件名}` → 決算書等の財務データを分析
- `/match {案件名}` → 最適な補助金を提案（財務データなしでも可）」

## エラー時

- 同名の案件フォルダが既に存在する → 上書きするか別名にするか確認
- ヒアリングが中断された → 「`/hearing {案件名}` で再開できます」と案内
