# 新規案件開始 & ヒアリング開始

新しい補助金申請支援案件を開始し、Phase 1 ヒアリングを実施します。

## 引数

$ARGUMENTS は不要（対話で案件名を取得）

## 前提条件

- なし（最初のステップ）

## 手順

### Step 1: 案件情報の取得

ユーザーに以下を確認する:
1. 案件名（会社名 or 略称）
2. 今日の日付を取得する

### Step 2: ワークスペース作成

```bash
mkdir -p "workspace/{案件名}"
```

以下のファイルを新規作成する（中身は案件名と日付で初期化）:

- `workspace/{案件名}/hearing_notes.md` — ヒアリング記録
- `workspace/{案件名}/company_data.md` — 会社・財務データ
- `workspace/{案件名}/strategy.md` — 補助金選定・戦略
- `workspace/{案件名}/draft_application.md` — 申請書ドラフト

各ファイルはヘッダー（`# {案件名} - {ファイル種別}`、作成日、ステータス）のみで初期化する。

### Step 3: 対応補助金の確認

`grants/` フォルダ内の補助金フォルダ一覧を確認し、ヒアリング時の参考にする。

### Step 4: ヒアリング開始

`.claude/skills/hearning-interview/SKILL.md` を読み込み、そのワークフローに従って Phase 1 を開始する。

**まず以下を1問だけ確認する:**

> 「ヒアリングの文字起こしや議事録データはありますか？
> ある場合はここに貼り付けてください。AI が内容を解析して情報を整理します。
> ない場合はそのまま対話形式で進めます。」

- **文字起こしあり** → SKILL.md のワークフローに従って解析・情報整理を実施
- **文字起こしなし** → 対話形式でヒアリングを開始

**重要ルール（共通）:**
- **1問ずつ質問する**（一度に大量の質問を投げない）
- 回答に応じて自然に深掘りする（事業内容・課題・投資計画の基本情報を収集する）
- 結果は `workspace/{案件名}/hearing_notes.md` に随時記録する
- 対面ヒアリングの場合、お客様に寄り添った平易な言葉で質問する
- どの補助金に申請するかの目星をつけながらヒアリングする
- `know-how/` フォルダを確認し、業種別・補助金別の知見を質問に反映する
- テンプレート固有の不足情報に対する逆質問は、`/add-template` 実行後に行う
- 文字起こし整理後や基本ヒアリング完了後に、不足項目の追加確認をそのまま続けて聞かない
- 不足項目は `hearing_notes.md` に「要確認」として記録し、その場では次ステップ案内で止める

### Step 5: ヒアリング完了判定

以下が全て揃ったら Phase 1 完了:
- [ ] 基本情報（会社名、業種、従業員数、年商）
- [ ] 事業の強み（3つ以上）
- [ ] 現状の課題（具体的に）
- [ ] 投資計画（何を、いくらで、なぜ）
- [ ] 将来ビジョン
- [ ] 補助金関連の確認事項（過去の申請歴、決算書の準備状況等）

この段階では、未取得や要確認の項目が残っていても `hearing_notes.md` に整理できていればよい。
テンプレート入力欄との差分確認と逆質問は `/add-template` で実施する。
したがって、Phase 1 完了時は追加質問を続けず、記録更新と `/analyze` または `/match` の案内を行う。

## 完了メッセージ

「ヒアリングが完了しました。次のステップ:
- `/analyze {案件名}` → 決算書等の財務データを分析
- `/match {案件名}` → 最適な補助金を提案（財務データなしでも可）」

## エラー時

- 同名の案件フォルダが既に存在する → 上書きするか別名にするか確認
- ヒアリングが中断された → 「`/hearing {案件名}` で再開できます」と案内
