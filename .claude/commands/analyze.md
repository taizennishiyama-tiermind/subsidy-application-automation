# 会社情報・財務分析（Phase 2）

決算書等の財務データを分析し、数値計画の基盤を作ります。

## 引数

$ARGUMENTS = 案件名（省略時はユーザーに確認する）

## 前提条件

- `workspace/{案件名}/hearing_notes.md` にヒアリング結果があること
- 決算書（PDF/Excel）や試算表があれば優先的に取り込むこと（なくても概算で進行可能）

## 手順

### Step 1: データ収集

1. `workspace/{案件名}/hearing_notes.md` からヒアリング結果を読み込む
2. **AI 側から最初に必ず次の質問を行う**

```text
財務分析を始めます。
決算書の PDF または Excel はありますか？

ある場合は、このチャットに添付してください。
AI がそのファイルをインプットとして読み取って分析します。

ない場合は、
- 「概算値で進める」
- 「一部だけ手入力で進める」
のどちらかを入力してください。
```

3. ユーザーに決算書等の財務データを提供してもらう（PDF/Excel）
4. **以下のどの方法で進めるかを最初に確定する**
   - 決算書PDFをAIが読み取って進める
   - 決算書Excel / 試算表ExcelをAIが読み取って進める
   - 一部資料＋不足分を手入力で補完して進める
   - 資料なしで概算値ベースで進める
5. PDFやExcelがある場合は、**AI側でその資料をインプットとして扱うことを明示してから**分析を開始する
6. 決算書がない場合はヒアリングの概算値を使用する（※ 概算値と注記）

### Step 2: 財務分析

`.claude/skills/application-financial-analysis/SKILL.md` のフレームワークに従い分析する:

0. **インプット方針の明記**
   - どの資料を使ったか
   - どの期間の数値を使ったか
   - 概算値が含まれるか
   を冒頭で整理する

1. **基礎指標の算出**
   - 付加価値額（営業利益 + 人件費 + 減価償却費）の3期推移
   - 給与支給総額の3期推移
   - 労働生産性（付加価値額 / 従業員数）

2. **4つの分析軸**
   - 収益性（営業利益率、経常利益率）
   - 安全性（自己資本比率、流動比率）
   - 成長性（売上成長率、利益成長率）
   - 人件費（人件費率、一人あたり給与）

3. **SWOT分析**
   - ヒアリングと財務データを統合して実施

### Step 3: 結果の記録

`.claude/skills/application-financial-analysis/SKILL.md` の「会社情報整理フォーマット」に従い、
`workspace/{案件名}/company_data.md` に記録する。

このとき、少なくとも以下を残す:

- 使用した資料の種類（PDF / Excel / 概算）
- AIがどの方法で入力処理したか
- 未取得または要確認の数値
- 申請書作成前に人が再確認すべき項目

## 完了メッセージ

「財務分析が完了しました。使用資料と未確定数値を整理済みです。次は `/match {案件名}` で最適な補助金をマッチングできます」

## エラー時

- 決算書がない → ヒアリングの概算値で分析し「※ 概算値。正式な決算書で要更新」と注記
- PDFが読めない → テキスト抽出可能なPDFか確認を依頼
- ヒアリング未実施 → `/new-case` または `/hearing` の実施を案内
