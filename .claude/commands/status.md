# 案件ステータス確認

案件の進捗状況をダッシュボード形式で表示します。

## 引数

$ARGUMENTS = 案件名（省略時は全案件を一覧表示）

## 手順

### 全案件一覧（引数なし）

1. `workspace/` 内のディレクトリ一覧を取得する
2. 各案件について以下を確認する:
   - `hearing_notes.md` の内容量 → ヒアリング進捗
   - `company_data.md` の内容量 → 財務分析進捗
   - `strategy.md` の内容量 → 補助金選定進捗
   - `draft_application.md` の内容量 → 申請書作成進捗
   - `application_data.json` の有無 → テンプレート書き込み準備

3. 以下の形式で表示する:

```
## 案件一覧

| 案件名 | Phase | ヒアリング | 分析 | マッチング | 申請書 | レビュー |
|--------|-------|-----------|------|-----------|--------|---------|
| A社    | 4     | done      | done | done      | 作成中  | -       |
| B社    | 1     | 進行中    | -    | -         | -      | -       |
```

### 個別案件の詳細（引数あり）

指定された案件の詳細ステータスを表示する:
1. 各ファイルの最終更新日時
2. ヒアリング済み項目のチェックリスト
3. 選定した補助金名（ある場合）
4. 次に実行すべきコマンド

## Phase の判定基準

| Phase | 判定条件 |
|-------|---------|
| 1 | hearing_notes.md が空 or 未完成 |
| 2 | hearing完了、company_data.md が空 |
| 3 | company_data完了、strategy.md が空 |
| 4 | strategy完了、draft_application.md が空 |
| 5 | draft完了、レビュー未実施 |
| 完了 | レビュー完了 or テンプレート書き込み完了 |

## エラー時

- workspace/ が空 → `/new-case` で案件を開始するよう案内
- 指定した案件が存在しない → 存在する案件の一覧を表示
