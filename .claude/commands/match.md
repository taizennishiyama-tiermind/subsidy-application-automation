# 補助金マッチング（Phase 3）

事業内容と会社状況から最適な補助金を提案します。

## 引数

$ARGUMENTS = 案件名（省略時はユーザーに確認する）

## 前提条件

- `workspace/{案件名}/hearing_notes.md` にヒアリング結果があること
- `workspace/{案件名}/company_data.md` があれば参照する（なくても実行可）

## 手順

### Step 1: データ読み込み

1. `workspace/{案件名}/hearing_notes.md` を読み込む
2. `workspace/{案件名}/company_data.md` があれば読み込む
3. `grants/` 配下の各補助金の `overview.md` を **全て** 読み込む

### Step 2: スコアリング

以下の6軸で各補助金を評価する（各10点満点、合計60点）:

| 評価軸 | 内容 |
|--------|------|
| 事業内容の適合性 | 補助金の対象事業と申請内容が合致するか |
| 従業員規模 | 対象者要件を満たすか、補助上限額はいくらか |
| 投資額 | 申請予定額が補助金の範囲に収まるか |
| 要件充足 | 付加価値額等の要件を満たせるか |
| スケジュール | 公募締切に間に合うか |
| 採択可能性 | ストーリーの組みやすさ、加点項目の取得見込み |

### Step 3: 結果提示

1. 推奨する補助金を優先度順に提示する
2. 各補助金について以下を説明する:
   - スコアの内訳
   - 適合理由
   - 想定される補助額
   - 注意点・リスク
3. ユーザーと補助金の選定を合意する

### Step 4: 記録

選定結果を `workspace/{案件名}/strategy.md` に記録する:
- 選定した補助金名
- 選定理由
- 想定スケジュール
- 今後の準備事項

## 完了メッセージ

「補助金が確定しました。次は `/draft {案件名}` で申請書の作成に進めます」

## エラー時

- grants/ フォルダに補助金がない → `/add-grant` での追加を案内
- ヒアリング未実施 → `/new-case` または `/hearing` の実施を案内
