# 補助金マッチング（Phase 3）

事業内容と会社状況から、**世の中にある補助金の中から**最適なものを提案します。
保存済みの補助金に限らず、Web検索で現在公募中の補助金を幅広く調査し、
**第一候補・第二候補・第三候補**として具体的な効果とともに提案します。

## 引数

$ARGUMENTS = 案件名（省略時はユーザーに確認する）

## 前提条件

- `workspace/{案件名}/hearing_notes.md` にヒアリング結果があること
- `workspace/{案件名}/company_data.md` があれば参照する（なくても実行可）

## 手順

### Step 1: 案件情報の読み込み

1. `workspace/{案件名}/hearing_notes.md` を読み込む
2. `workspace/{案件名}/company_data.md` があれば読み込む
3. `.claude/skills/subsidy-matcher/SKILL.md` を読み込む
4. ヒアリング情報から以下を整理する:
   - 業種・事業内容
   - 従業員規模（小規模事業者か中小企業か）
   - 投資内容・金額
   - 課題の種類（DX推進 / 設備投資 / 販路開拓 / 新事業 / 省エネ等）
   - 所在地（都道府県・市区町村）

### Step 2: 保存済み補助金の評価（第1段階）

1. `grants/` 配下の各補助金の `overview.md` を **全て** 読み込む
2. `grants/` 配下の `docs/` にPDFがあれば内容を読み込む（overview.mdより優先）
3. `subsidy-matcher/SKILL.md` のスコアリングフレームワーク（6軸・各10点）で評価する

### Step 3: Web検索による追加調査（第2段階）

`subsidy-matcher/SKILL.md` の検索プロセスに従い、以下を実施する:

1. 業種・課題・地域に合わせた検索クエリを設計する
2. 経産省・中小企業庁・J-Net21・都道府県公式サイト等を検索する
3. 見つかった補助金を保存済みと同じ6軸でスコアリングする
4. 公式URLで申請期間・補助率・要件が最新であることを確認する

### Step 5: ユーザーとの合意

提示した候補についてユーザーの意向を確認し、補助金を確定する。

### Step 6: 記録と補助金蓄積

1. 選定結果を `workspace/{案件名}/strategy.md` に記録する

2. **Web検索で見つかった有力な補助金が `grants/` 未登録の場合:**
   - 「次回の案件でも使えるように登録しましょうか？」とユーザーに提案する
   - 同意を得たら `/add-grant {補助金名}` の実行を案内する（または自動実行）

### Step 7: 申請準備ガイドの表示（補助金確定後に必ず実行）

補助金が確定したら、**その補助金の申請に必要な準備をまとめて提示する**。

#### 7-1. 蓄積情報の参照

以下の優先順位で情報を参照する:

1. `grants/{補助金名}/docs/` にPDFがあれば読み込む（公募要領・交付規定等）
2. `grants/{補助金名}/overview.md` を読み込む
3. `grants/{補助金名}/templates/README.md` を読み込む（テンプレートの入手先）
4. `grants/{補助金名}/templates/` を確認する（登録済みテンプレート実体の有無）

#### 7-3. 表示後の待機

申請準備ガイドを表示したら、ユーザーがテンプレートを添付するまで待機する。

- テンプレートが添付されたら → 自動的に Step 5（`/add-template`）を開始する
- 「既に持っている」と入力されたら → `/add-template {補助金名}` のコマンド実行を案内する
- 「後でやる」と入力されたら → 「準備できたらテンプレートを添付するか `/add-template {補助金名}` を実行してください」と案内して終了

## エラー時

- ヒアリング未実施 → `/new-case` または `/hearing` の実施を案内
- Web検索で情報が取得できない → 保存済み補助金のみでスコアリングし「Web検索できませんでした」と注記
- 申請締切が1週間以内 → ⚠️マークで強調し、次回公募への切り替えも提案
- 申請準備ガイドの情報が不足（grants/ 未登録・docs/ なし）→ Web検索で補い、「詳細は公式サイトで確認してください」と付記
