# テンプレートファイルの登録・解析・情報充足（Step 5 / 必須）

補助金の申請書テンプレート（Excel/Word）を登録・解析し、
テンプレートの設問に対してヒアリング情報が十分かを確認・補完します。

**このステップはドラフト作成前の必須工程です。**
テンプレートがないと申請書の精度・自動書き込みが保証できないため、
テンプレートが受け取れるまでドラフトへは進みません。

## 引数

$ARGUMENTS = 補助金名（省略時はStep 0で確認）

---

## 手順

### Step 0: 前提確認と補助金フォルダの確認

1. 補助金名が引数で指定された場合:
   - `grants/{補助金名}/` フォルダが存在するか確認する
   - 存在しない → 「先に `/add-grant {補助金名}` を実行してください」と案内して終了

2. 補助金名が指定されていない場合:
   - 現在進行中の案件の `strategy.md` を確認し、選定補助金を自動検出する
   - 検出できた場合は「{補助金名} のテンプレートを登録します」と確認して進む
   - 検出できない場合はユーザーに1問で確認する

3. 既存登録の確認:
   - `grants/{補助金名}/templates/` 配下にサブフォルダが存在し、その中にテンプレート実体と `TEMPLATE_SKILL.md` がある場合は登録済み内容を表示する:
     ```
     📋 既存登録あり: {ファイル名}（{年度}、登録日: {日付}）
     別バージョンを追加登録しますか？ [はい / スキップして差分確認へ]
     ```
   - 「スキップ」の場合は Step 4（差分分析）から再開する

### Step 1: テンプレートファイルの受け取り

以下のメッセージを表示する:

```
📂 Step 5: テンプレートファイルの登録（必須）

補助金の公式テンプレートファイル（.xlsx / .docx）を
以下のフォルダにドラッグ＆ドロップしてください。

  📁 grants/{補助金名}/templates/

【手順】
  1. VSCode のエクスプローラーで上記フォルダを開く
  2. テンプレートファイルをそのフォルダ内にドラッグ
  3. 配置できたら「完了」と送信してください

【入手先】
  テンプレートをお持ちでない場合は
  grants/{補助金名}/templates/README.md に公式URLが記載されています。
```

**「完了」と送信された場合:**
1. `grants/{補助金名}/templates/` 配下に新たに配置された `.xlsx` / `.docx` ファイルを検出する
2. ファイルが見つかった場合:
   - ファイル名（拡張子なし）をフォルダ名としてサブフォルダを作成する
   - `grants/{補助金名}/templates/{ファイル名（拡張子なし）}/{ファイル名}` に移動する
   - 以下を表示して Step 2 へ進む:
     ```
     📂 テンプレートを格納しました
     → grants/{補助金名}/templates/{ファイル名（拡張子なし）}/{ファイル名}
     TEMPLATE_SKILL.md と必要な運用メモをこのフォルダ専用に生成します。
     解析を開始します...
     ```
3. ファイルが見つからない場合:
   ```
   ⚠️ ファイルが確認できませんでした。
   grants/{補助金名}/templates/ フォルダにファイルが配置されているか確認し、
   もう一度「完了」と送信してください。
   ```
   → 再び待機する。

**「持っていない」と入力された場合:**
```
⚠️ テンプレートは申請書作成の必須資料です。

【入手方法】
grants/{補助金名}/templates/README.md に記載の公式URLからダウンロードしてください。

ダウンロード後、以下のフォルダにドラッグして「完了」と送信してください。
  📁 grants/{補助金名}/templates/

（ドラフト作成はテンプレート登録後に進めます）
```
→ 「完了」が送信されるまで待機する。ドラフトへは進まない。

### Step 2: バージョン情報の確認（1問のみ）

```
📋 1点だけ確認させてください。

このテンプレートは何年度（または何次公募）のものですか？
わからない場合は「不明」で構いません。
（例: 2026年度、第23次、令和6年度）
```

回答を受け取ったら即座に Step 3 へ進む（それ以上は聞かない）。

### Step 3: テンプレートの自動解析

ファイル拡張子を確認してから、形式に応じた解析を実行する。

**共通ルール:**
- 1シート目だけで判断しない
- Excel は全シート、Word は全セクション・全テーブル・全段落を必ず確認する
- 「記入例」「説明」「見本」「注意事項」も入力ルール把握のため必ず参照する
- 直接入力が必要な箇所、別シートから自動反映される箇所、数式セル、参照専用欄を分けて整理する
- 最終的に、**申請に提出するシートの入力箇所**は全て特定する
- **「記入例」「記入方法」「説明」シートは参照専用として扱い、原則として書き込み対象にしない**

#### Excel（.xlsx / .xls）の場合

```bash
.venv/bin/python "grants/{補助金名}/scripts/fill_template.py" analyze \
  --template "grants/{補助金名}/templates/{ファイル名（拡張子なし）}/{ファイル名}"
```

スクリプトが利用できない場合はファイルを直接読み込み、以下を把握する:
- 全シートの確認と役割整理
- 入力対象シートの特定
- 「記入例」「説明」「見本」シートに書かれたルール、記入順、禁止事項、参照関係の把握
- 提出対象シートと参照専用シートの切り分け
- 各シートのラベルセルと入力セルのペア（結合セルは左上セルが書き込み対象）
- ドロップダウン等の特殊セルの把握
- 設問の文言と記入意図
- プレースホルダーや Named Range が無い場合でも、表構造、周辺ラベル、結合セル、数式セル、行間隔から入力位置を推定する
- 2行1組や複数列ブロックの表は、その構造ごとにマッピング候補を作る
- シート間参照関係を把握し、どのシートに元データを入れるべきかを特定する
- 固定セル候補が既に分かっていても、そのまま信用せず、**今回のテンプレート版でラベルアンカーと周辺構造から差し込み先を再確認する**
- 必須欄は「どこに入れるか」を必ず特定し、未解決なら未解決として明示して次へ流さない

解析結果の形式:
```
【Excelテンプレート解析結果】
確認シート数: {N}シート
必須入力項目数: {N}項目
入力対象シート: {シート名1}, {シート名2}, ...
参照専用シート: {シート名...}
提出対象シート: {シート名...}
主要入力項目:
  - {ラベル1} → シート:{シート名} セル:{B3}（必須）
  - {ラベル2} → シート:{シート名} セル:{B5}（必須）
  ...
```

#### Word（.docx）の場合

```bash
.venv/bin/python "grants/{補助金名}/scripts/fill_template.py" analyze \
  --template "grants/{補助金名}/templates/{ファイル名（拡張子なし）}/{ファイル名}"
```

スクリプトが利用できない場合はファイルを直接読み込み、以下を把握する:
- 全段落、全見出し、全テーブルの確認
- `{{プレースホルダー}}` 形式のトークンを全て検出
- ブックマーク（Word Bookmark）が設定されている場合はその名前と位置を把握
- 表（テーブル）内のセルと対応するラベルの関係
- 各設問の文字数制限・記入欄サイズ（テキストボックスや表の行高から推定）
- 見出し構造（どのセクションに何を書くか）
- プレースホルダーが無い場合でも、見出し、表ラベル、段落順、欄サイズから「どこに何を書くか」を推定する
- 単なる参考様式や記載項目一覧の場合は、その設問順を完成版ドラフトの出力順として採用する
- 参照説明や記入例の段落も確認し、入力方針へ反映する

解析結果の形式:
```
【Wordテンプレート解析結果】
確認対象: 全段落 / 全テーブル / 全セクション
必須入力項目数: {N}項目
入力方式: プレースホルダー / ブックマーク / 表セル（いずれか）
主要入力項目:
  - {{company_name}} → 会社名（必須）
  - {{project_title}} → 事業名（必須）
  - 第2表 / 行3 / 列2 → 事業費合計（必須）
  ...
```

#### PDF の場合

PDF は書き込みができないため、以下を案内して終了する:
```
⚠️ PDFは自動書き込みに対応していません。
  Excel または Word 形式のテンプレートをご用意ください。
  PDF が公募要領の場合は grants/{補助金名}/docs/ に格納して参照用として使用します。
```

### Step 4: ヒアリング情報との差分分析

1. 以下のファイルを読み込む:
   - `workspace/{案件名}/hearing_notes.md`
   - `workspace/{案件名}/company_data.md`
   - `workspace/{案件名}/strategy.md`
   - `.claude/skills/grant-application-writer/SKILL.md`

2. `grant-application-writer` の設問解釈ルールに従い、各入力欄がどの審査観点を見ているかを整理してから、テンプレートの必須入力項目を一つひとつチェックし、ヒアリング情報で回答できるかを判定する:

   | 判定 | 条件 |
   |------|------|
   | ✅ 充足 | ヒアリング記録に明示的な記載あり |
   | ⚠️ 推測 | 関連情報から推測可能だが明示なし |
   | ❌ 不足 | ヒアリング記録に記載なし・確認要 |

3. 差分一覧を作成する（内部保持、次のステップで使用）
4. 全シート分の入力項目一覧に対して抜け漏れがないか再確認する
5. **提出対象シートの必須欄が何かを明示し、参照専用シートは書き込み対象から除外する**
6. **必須欄ごとに、セル番地だけでなく「どのラベル根拠でその位置と判断したか」を保持する**

---

### Step 5: 不足情報の逆質問（情報充足）

不足 / 推測の項目について、**1問ずつ順番に**ユーザーへ確認する。

まず差分のサマリーを表示する:
```
📊 テンプレート解析とヒアリング情報の照合結果

  ✅ 充足済み: {N}項目
  ⚠️ 推測で補完予定: {N}項目
  ❌ 追加確認が必要: {N}項目

追加確認が必要な項目を1つずつ伺います。
```

**不足項目ごとに以下の形式で質問する（1問ずつ）:**
```
❓ 質問 {i}/{N}：{テンプレートのラベル文言}

テンプレートの {シート名} シートの「{ラベル}」欄に記入が必要です。
{設問の意図の補足説明}

→ {具体的な回答例を示す}
```

回答を受け取ったら即座に次の質問へ進む。全質問が完了したら Step 6 へ。

**推測項目は自動補完し確認を取る:**
```
⚠️ 推測補完: {ラベル}
  ヒアリング内容から「{推測値}」と判断しました。
  修正が必要な場合は入力してください。（そのままでよければ「OK」）
```

### Step 7: TEMPLATE_SKILL.md の自動生成

解析結果と充足済み情報をもとに `grants/{補助金名}/templates/{ファイル名（拡張子なし）}/TEMPLATE_SKILL.md` を自動生成する。
このファイルは**このテンプレート専用の知見ファイル**であり、他の補助金・他のテンプレートのフォルダには絶対に生成しない。

**生成ルール（共通）:**
1. 変数キーを自動命名（ラベルから英語スネークケースを生成）
2. 必須 / 任意の判定を記載
3. 逆質問で得た補完情報を各変数の `default_value` として記録
4. プレースホルダーが無い場合でも、AI が推定した入力位置・推定根拠・手動確認ポイントを記録する
5. 全シート確認済みであることを明記する
6. 入力対象だけでなく、参照専用シートや自動計算シートの扱いも記録する

**Excelテンプレートの場合:**
- 全シートを確認したうえで分類結果を記録
- 入力セルとラベルのペアを整理
- どのシートのどのセルが元入力セルかを優先して記録

**Wordテンプレートの場合:**
- プレースホルダートークン（`{{key}}`）またはブックマーク名を変数キーとして使用
- 表セルの場合は「テーブルインデックス / 行 / 列」形式で位置を記録
- プレースホルダーが無い場合は、見出し / テーブル番号 / 行列 / 段落順を位置情報として記録する

## エラー時

| エラー | 対処 |
|--------|------|
| Python環境がない | `bash scripts/setup.sh` を実行するよう案内 |
| テンプレートが見つからない | ファイルパスと拡張子（.xlsx/.docx）を確認するよう案内 |
| 解析エラー（ファイルが壊れている） | 公式サイトから再ダウンロードするよう案内 |
| セルマッピングがずれる | TEMPLATE_SKILL.md を確認し、該当セルを手動修正するよう案内 |
| テンプレートが入手できない | 公式サイトのURLを案内し、受け取るまで次のステップへは進まない |
