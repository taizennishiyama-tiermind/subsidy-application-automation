# 申請書レビュー（Phase 5）

作成した申請書を審査基準に照らしてセルフレビューし、品質を高めます。

## 引数

$ARGUMENTS = 案件名（省略時はユーザーに確認する）

## 前提条件

- `workspace/{案件名}/draft_application.md` が作成済みであること

## 手順

### Step 1: レビュー準備

1. `workspace/{案件名}/draft_application.md` を読み込む
2. `workspace/{案件名}/strategy.md` から選定した補助金名を取得する
3. `grants/{補助金名}/overview.md` の審査基準を読み込む
4. `grants/{補助金名}/docs/` フォルダを確認し、PDFファイルがあれば内容を読み込む
   - 公募要領: 審査基準の詳細・必須記載項目の最終確認・ページ数制限
   - 交付規定: 対象外経費の記載がないか確認
   - Q&A: 誤りやすいポイントのチェック
   - `docs/` の内容と `overview.md` に差異がある場合は `docs/` を優先する
5. `.claude/skills/reviewer/SKILL.md` のチェック観点と改善提案方針を読み込む
6. `know-how/` フォルダにファイルがあれば読み込み、自社独自の評価観点を追加する

### Step 2: 4観点レビュー

#### 形式チェック
- [ ] ページ数制限を守っているか
- [ ] 必須記載項目を全てカバーしているか
- [ ] 図表が適切に配置されているか（案として）

#### 内容チェック
- [ ] ストーリーに一貫性があるか（現状→課題→解決→効果）
- [ ] 自社の強みが明確に伝わるか
- [ ] 課題が具体的（数値データ付き）か
- [ ] 解決策と課題の因果関係が論理的か
- [ ] 革新性/新規性が明確か

#### 数値チェック
- [ ] 全ての数値に根拠があるか
- [ ] 付加価値額の要件を満たすか（年平均+3%以上）
- [ ] 給与支給総額の要件を満たすか（年平均+2%以上）
- [ ] 最低賃金の要件を確認したか
- [ ] 売上計画と利益計画が整合しているか

#### 審査基準対応チェック
- [ ] 技術面/事業面の審査項目に全て言及しているか
- [ ] 政策面の審査項目に対応しているか
- [ ] 加点項目を最大限活用しているか
- [ ] `docs/` の公募要領に記載の評価ポイントに漏れなく対応しているか

### Step 3: レビュー結果の出力

以下の形式で結果を提示する:

```
## レビュー結果サマリー

### 総合評価: A（採択見込み高）/ B（修正で改善可）/ C（大幅な修正が必要）

### 改善必須項目（CRITICAL）
1. ...

### 改善推奨項目（RECOMMENDED）
1. ...

### 良い点
1. ...

### 添付書類チェックリスト
- [ ] ...
```

### Step 4: 修正の実施

CRITICAL項目がある場合:
1. 修正内容をユーザーに提案する
2. 承認を得たら `draft_application.md` を修正する
3. 修正後に再レビューする

## 完了メッセージ

「レビューが完了しました。次のステップ:
- `/fill {案件名}` → Excel/Wordテンプレートへの書き込み
- 修正が必要な場合は、修正後に再度 `/review {案件名}` を実行してください」

## エラー時

- draft_application.md が存在しない → `/draft` の実施を案内
- 補助金が未選定 → `/match` の実施を案内
